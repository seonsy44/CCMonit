# docs 운영 규약

## 원칙

- `docs/`는 LLM이 유지·관리하는 프로젝트 wiki다.
- 인간은 방향을 설정하고 검토한다. LLM은 문서의 작성, 갱신, 상호 참조, 정리를 담당한다.
- 명세는 **"무엇을"** 구현할지를 기술한다. **"어떻게"** 구현할지는 기술하지 않는다.
- 파일을 잘게 분리하여 AI가 필요한 것만 읽을 수 있도록 한다. 하나의 파일에 모든 내용을 넣지 않는다.

## 페이지 종류

| 종류 | 위치 | 설명 |
|------|------|------|
| 개요 | `overview.md` | 제품 개요, MVP 범위, 확장 계획 |
| 용어집 | `glossary.md` | 용어 정의 |
| 기능 명세 | `features/*.md` | 기능별 명세. 하나의 기능 = 하나의 파일 |
| 화면 명세 | `screens/*.md` | 화면별 명세. 하나의 화면 = 하나의 파일 |
| 운영 규약 | `conventions.md` | 이 문서. docs 관리 방법 |

각 카테고리 폴더(`features/`, `screens/`)에는 해당 카테고리의 `index.md`가 있다.

## index.md

`docs/index.md`는 전체 문서의 카탈로그다.

- 모든 페이지를 링크와 한 줄 설명으로 등록한다.
- 카테고리별로 그룹핑한다.
- 페이지를 추가하거나 삭제하면 **반드시** index.md를 갱신한다.
- AI는 작업 시작 시 index.md를 먼저 읽어 관련 페이지를 찾는다.

## log.md

`docs/log.md`는 문서 변경의 시간순 기록이다.

- 형식: `## [YYYY-MM-DD] 동작 | 대상`
- 동작 예시: `init`, `add`, `update`, `delete`, `lint`
- 문서를 변경할 때마다 **반드시** 항목을 추가한다 (append-only).

## 문서 변경 절차

### 페이지 추가

1. 적절한 위치에 마크다운 파일을 생성한다.
2. 해당 카테고리의 `index.md`에 항목을 추가한다.
3. `docs/index.md`에 항목을 추가한다.
4. 관련 페이지에 상호 참조 링크를 추가한다.
5. `docs/log.md`에 기록한다.

### 페이지 수정

1. 해당 파일을 수정한다.
2. 수정으로 인해 다른 페이지의 참조가 달라지면 함께 갱신한다.
3. index.md의 설명이 달라졌으면 갱신한다.
4. `docs/log.md`에 기록한다.

### 페이지 삭제

1. 해당 파일을 삭제한다.
2. 해당 카테고리의 `index.md`에서 항목을 제거한다.
3. `docs/index.md`에서 항목을 제거한다.
4. 다른 페이지에서 해당 파일을 참조하는 링크를 제거하거나 갱신한다.
5. `docs/log.md`에 기록한다.

## 상호 참조

- 관련 페이지 간에는 상대 경로 링크로 연결한다. 예: `[settings.md](./settings.md)`
- 기능 명세에서 화면 명세를, 화면 명세에서 기능 명세를 참조할 수 있다.
- 고아 페이지(어디서도 참조하지 않는 페이지)가 없도록 한다.

## 헬스체크 (lint)

주기적으로 다음을 점검한다:

- index.md에 등록되었으나 실제 파일이 없는 항목 (깨진 링크)
- 파일은 존재하나 index.md에 등록되지 않은 페이지 (누락)
- 페이지 간 상호 참조 링크의 유효성
- 내용 간 모순이나 불일치
- 용어집에 정의되지 않은 새로운 용어
