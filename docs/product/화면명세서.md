# Claude Code Monitor 화면명세서

## 1. 화면 설계 방향
화면은 “운영 중 빠른 파악”이 핵심이므로 **한 화면에서 상태, 활동, 소모, 이상 징후를 동시에 볼 수 있는 대시보드형 TUI** 로 설계한다.

예시 이미지처럼 박스형 패널 레이아웃을 유지하되, 단순 나열이 아니라 **위는 세션 요약, 가운데는 실행 구조, 아래는 세부 활동** 흐름을 따른다.

---

## 2. 전체 화면 목록
1. 메인 대시보드
2. 태스크 상세 화면
3. 세션 리포트 화면
4. 토큰 분석 화면
5. 알림/이상 징후 화면
6. 설정/어댑터 상태 화면
7. 최근 세션 목록 화면

---

## 3. 메인 대시보드

### 3.1 목적
현재 세션에서 “무슨 일이 벌어지고 있는가”를 즉시 파악하는 기본 화면

### 3.2 주요 구성

#### A. 헤더 영역
표시 항목:
- 제품명 / 세션명
- 현재 시각
- 세션 ID
- 모델명
- 작업 디렉터리
- 총 경과 시간
- idle 시간
- 전체 토큰 요약
- 어댑터 상태(ex: Runtime OK / FileWatch OK / Token Estimator Fallback)

#### B. 상단 요약 바
표시 항목:
- 활성 에이전트 수
- 활성 태스크 수
- 완료 태스크 수
- 실패 태스크 수
- 경고 수
- 파일 변경 수
- 전체 툴 호출 수

#### C. Tools 요약 패널
표시 항목:
- 툴별 호출 수
- 최근 1분/전체 누적 토글
- input/output 토큰 합계(가능한 경우)
- 오류 발생 툴 강조

#### D. Subagents / Agents 패널
표시 항목:
- 에이전트 이름 또는 역할
- 상태 아이콘
- 실행 시간
- 현재 태스크 요약
- 토큰 사용량
- 마지막 이벤트 시각

#### E. Skill 패널
표시 항목:
- 현재/최근 실행 스킬 목록
- 스킬 시작 시각
- 소속 task/agent
- status
- 해당 스킬에서 발생한 툴 수

#### F. Teams 패널
표시 항목:
- 팀명
- 팀 상태
- 포함된 agent 수
- 진행 중 태스크 수
- 누적 토큰

#### G. Tasks 패널
표시 항목:
- 최근 또는 활성 태스크 목록
- 상태
- 소속 agent/team
- 실행 시간
- retry/error 개수
- 토큰 사용량

#### H. File Activity 패널
표시 항목:
- 시각
- 파일 경로
- create/update/delete
- 관련 task/agent
- 최근 변경 강조

#### I. Footer / Hotkey 영역
표시 항목:
- 키보드 단축키 안내
- refresh 주기
- 현재 필터/정렬 상태

### 3.3 기본 레이아웃 제안
- 상단: 헤더 + 요약 바
- 중단 좌측: Tools / Agents / Skills
- 중단 우측: Teams / Tasks
- 하단: File Activity + Alerts strip + Footer

### 3.4 상태 표현 가이드
- running: 강조 표시
- waiting: 중간 강조
- done: 약한 톤
- failed: 경고 색/심볼
- stuck 후보: 별도 배지
- estimated token: 물결(~) 또는 EST 배지

### 3.5 빈 상태
- 세션 없음: “대기 중” 안내
- 팀 없음: “(no teams)”
- 태스크 없음: “(no active tasks)”
- 파일 변경 없음: “watching for changes...”

## 4. 태스크 상세 화면

### 4.1 목적
특정 태스크 하나를 깊게 분석하는 화면

### 4.2 표시 항목
- 태스크 제목 / ID / 상태
- 상위 agent / team / parent task
- 시작/종료/경과 시간
- 토큰 상세(input/output/cache/total)
- 관련 스킬 목록
- 툴 호출 타임라인
- 최근 로그
- 파일 변경 목록
- warnings / errors / retries
- 최종 summary 또는 결과 메모

### 4.3 인터랙션
- Enter: 선택 태스크 상세 진입
- t: 토큰 탭
- g: 로그 탭
- p: 파일 탭
- h: 상위 계층 보기

---

## 5. 세션 리포트 화면

### 5.1 목적
진행 중 또는 종료된 세션을 요약해서 회고하는 화면

### 5.2 표시 항목
- 세션 총 시간
- 총 에이전트/태스크 수
- 완료/실패 비율
- 토큰 총합
- 상위 토큰 소비 작업 Top N
- 장시간 작업 Top N
- 오류 많은 작업 Top N
- 파일 변경 상위 목록
- 툴 사용량 분포

### 5.3 부가 기능
- Markdown/JSON/CSV export
- 세션 간 비교(후속 버전)

---

## 6. 토큰 분석 화면

### 6.1 목적
토큰을 “어디서, 왜, 얼마나” 썼는지 분석하는 화면

### 6.2 표시 항목
- 전체 input/output/cache read/cache write
- agent별 토큰 사용량
- task별 토큰 사용량
- tool별 토큰 사용량
- exact/estimated 비율
- 시간 대비 토큰 사용량
- 파일 1개 변경당 토큰 사용량

### 6.3 추천 추가 표시
- 급증 구간 하이라이트
- 재시도와 토큰 증가의 상관관계
- 큰 컨텍스트를 사용한 작업 순위

---

## 7. 알림/이상 징후 화면

### 7.1 목적
놓치기 쉬운 문제를 별도로 모아보는 화면

### 7.2 표시 항목
- stuck 후보 태스크
- stderr 연속 발생 작업
- 토큰 급증 이벤트
- orphan 이벤트
- adapter 장애
- 동일 파일 경쟁 수정 경고

### 7.3 인터랙션
- Enter: 원인 태스크로 이동
- m: mute
- c: clear/resolved 처리

---

## 8. 설정/어댑터 상태 화면

### 8.1 목적
수집 상태와 로깅 정책을 관리하는 화면

### 8.2 표시 항목
- refresh interval
- 저장 경로
- 로그 레벨
- 민감정보 마스킹 설정
- 토큰 추정기 사용 여부
- adapter별 상태/마지막 heartbeat/오류 메시지

---

## 9. 최근 세션 목록 화면

### 9.1 목적
이전 세션을 회고 모드로 다시 열기 위한 화면

### 9.2 표시 항목
- 세션 ID
- 시작 시각
- 총 시간
- 모델
- 토큰 총합
- 실패 수
- export 파일 존재 여부

---

## 10. 화면 전환 구조
- 메인 대시보드 → 태스크 상세
- 메인 대시보드 → 토큰 분석
- 메인 대시보드 → 알림 화면
- 메인 대시보드 → 설정 화면
- 메인 대시보드 → 최근 세션 목록
- 최근 세션 목록 → 세션 리포트(read-only)

## 12. 디자인 세부 가이드
- 기본은 다크 테마 + 낮은 채도 강조색
- 패널 경계는 명확하되 과도한 장식은 지양
- 숫자와 상태는 세로 정렬이 잘 맞아야 한다
- 너무 긴 파일 경로는 앞부분 축약 또는 중간 생략 처리
- 화면 폭이 좁을 때는 하단 패널부터 접힘 가능해야 한다
- 핵심 숫자(토큰, 경과 시간, 오류)는 빠르게 비교 가능해야 한다

---

## 13. 반응형/터미널 크기 대응

### 넓은 화면
- 다중 패널 배치
- Tools/Agents/Tasks/FileActivity 동시 표시

### 좁은 화면
- 패널 탭 전환 방식
- 핵심 요약 + 선택 패널 집중 표시

### 매우 좁은 화면
- 단일 컬럼 리스트 모드
- 헤더 + 현재 활성 작업 우선
