# dev-sprint

이 스킬은 **스프린트 1개를 처음부터 끝까지 수행하는 메인 실행 스킬**이다. `dev-open`에서 확정한 스프린트 목록 중 하나를 받아 실행한다.

## 입력

- `$ARGUMENTS`에 이번 스프린트 대상을 짧게 준다.
- 예: `domain agent entity`, `README cleanup`, `docs link repair`

## scratch 경로 인지

`dev-open`에서 명시한 scratch 경로를 그대로 사용한다. 별도로 scratch를 읽거나 쓰지 않는다. 범위 초과 또는 열린 질문이 생기면 `dev-close`가 올바른 경로에 반영한다.

## 먼저 읽을 것

- 스프린트 대상 관련 docs 파일 (어떤 파일인지 모를 때는 `docs/index.md` 먼저 확인)
- 관련 코드 파일

> 참고: `.claude/context/*.md`는 `dev-open`에서 이미 로드됨. 새 세션에서 직접 시작할 경우에만 읽는다.

## 해야 할 일

1. 이번 스프린트 범위를 1문장으로 확인한다.
2. 불확실성이 높으면 범위를 한 단계 줄인다. 낮으면 `dev-open`이 정한 범위를 유지한다.
3. 실제 코드 또는 문서 작업을 수행한다.
4. 완료 후 변경 파일 목록을 정리한다.
5. 목표 대비 과도한 수정이 없는지 짧게 확인한다.
6. **커밋 메시지 한 줄을 추천한다** (사용자가 커밋 여부를 결정한 뒤 다음 스프린트로 이동).
7. 세션 전체 stop line에 비추어 다음 스프린트 진행 여부를 표시한다.

### 범위 초과 시

스프린트 수행 중 예상보다 범위가 크다고 판단되면:
- 현재 진행한 만큼만 마무리하고 커밋 메시지를 추천한다.
- 남은 부분은 open-question 또는 다음 세션 스프린트로 표시한다 (`dev-close`가 올바른 scratch 경로에 반영).
- `dev-close`로 이동할 것을 제안한다.

## 출력 형식

- 이번 스프린트 범위 확인
- 수행한 작업 요약
- 수정 파일 목록
- 남은 리스크 / 참고사항
- **추천 커밋 메시지**: `type: 한 줄 설명`
- 다음 스프린트 진행 여부 (계속 / `dev-close`로 이동)
