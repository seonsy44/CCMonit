# dev-check

이 스킬은 **optional 체크포인트**다. `dev-sprint` 사이에 진행 상황을 점검하고 남은 스프린트 계획을 조정한다.

**언제 쓰나:**

- 스프린트 3개 이상인 세션에서 중간 점검이 필요할 때
- `dev-sprint` 중 예상 밖 상황이 생겨 계획 재조정이 필요할 때
- 다음 스프린트를 시작하기 전에 방향을 확인하고 싶을 때

2개 이하 스프린트이면 불필요하다.

## 해야 할 일

1. `dev-open`에서 확정한 스프린트 목록과 현재 완료 상태를 대조한다.
2. 세션 stop line에 비추어 남은 스프린트가 여전히 적절한지 판단한다.
3. 계획 변경이 필요하면 제안한다:
   - 순서 변경
   - 스프린트 범위 축소 / 다음 세션으로 이월
   - 예상 밖 발견 사항에 따른 추가 스프린트

## 출력 형식

- 현재 개발건
- 완료된 스프린트
- 남은 스프린트 목록 (변경 시 조정 내용 포함)
- 계속 진행 여부 / 다음 `dev-sprint` 대상
