# dev-pause

이 스킬은 **현재 개발건을 완료하지 않은 채 다른 개발건으로 전환**할 때 쓴다.
dev-close와 달리 개발건을 dev-list에서 제거하지 않고, 이어받기에 필요한 최소 컨텍스트만 기록한다.

**언제 쓰나:**
- dev-open 후 스프린트를 일부만 진행하고 다른 dev로 전환해야 할 때
- 긴급한 다른 작업이 생겨서 현재 dev를 나중에 이어해야 할 때

**언제 안 쓰나:**
- 개발건이 완결됐으면 `dev-close`를 쓴다
- 같은 세션에서 이어서 진행하면 그냥 `dev-sprint`를 쓴다

## 해야 할 일

1. 이번 세션에서 완료한 스프린트를 정리한다.
2. 남은 스프린트를 정리한다.
3. 이어받기에 필요한 핵심 컨텍스트를 1~3줄로 적는다:
   - 어떤 파일을 먼저 읽으면 되는지
   - 어떤 결정이 이미 내려졌는지
   - 주의할 점
4. `.claude/scratch/pause-{slug}.md`에 기록한다.
5. `dev-list.md` 예상 스프린트 컬럼을 남은 스프린트로 갱신한다.

## pause 파일 형식

```
# Pause: {slug}
날짜: {YYYY-MM-DD}
완료: {완료한 스프린트}
다음: {남은 스프린트 중 첫 번째}
컨텍스트:
- {이어받기 핵심 정보 1~3줄}
```

## 출력 형식

- 현재 개발건: {slug}
- 완료 스프린트
- 남은 스프린트
- 이어받기 컨텍스트 (1~3줄)
- 생성한 파일: `.claude/scratch/pause-{slug}.md`
