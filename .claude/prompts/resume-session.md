## 이번 세션 원칙
- 이전 세션의 남은 작업 중 가장 작은 단위 하나만 선택한다.
- 이미 끝난 일은 반복하지 않는다.
- 구현, 문서, 검증 범위를 분리해서 본다.
- 세션 끝에 handoff가 가능하도록 기록 포맷까지 제안한다.

## 원하는 출력
1. 현재 상태 요약
2. 이번 세션에서 할 일 1개
3. 수정할 가능성이 높은 파일
4. 사용할 skill 또는 절차
5. 세션 종료 시 남길 handoff 내용 초안

## 추가 지시
작업 범위가 크면 더 쪼개고, 이번 세션에서 다루지 않을 것은 명시적으로 제외해줘.
